UPDATE 08 — Process-Level Hardening (security.js)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
ගොනු: security.js (නව ගොනු), index.js, mongodb.js

ගැටළුව:
  Server ඇතුළෙ child_process module ගෙන් shell commands
  (exec, spawn, fork) run කිරීමෙන් reverse shell, server
  takeover කළ හැකිය. Process.env හරහා credentials expose
  විය හැකිය.

සකස් කළ දේ:

  Child Process Block:
    exec, execSync, spawn, spawnSync, execFile, execFileSync,
    fork — ඔක්කොම Object.defineProperty ගෙන් freeze + override
    කළා. Call කළොත් error throw. Module freeze කළා.

  Env Var Deletion:
    MONGODB_URI සහ UNBAN_CODE startup ෙදී closed-scope
    constants ලෙස store කරලා process.env ෙකොන් delete කළා.
    Code ගේ ඕනෑ ම තැනකදී process.env.MONGODB_URI read
    කළොත් undefined. Accessor functions ගෙන් ම ගත හැකිය.

  Console.error Redaction:
    console.error patch කළා. Error log ඇතුළේ sensitive
    value appear වුණොත් [REDACTED] ලෙස replace කරනවා.

  Secure Error Handler Middleware:
    Express global error handler — error messages redact
    කරලා client ට send කරනවා. Stack traces expose නොවෙ.
